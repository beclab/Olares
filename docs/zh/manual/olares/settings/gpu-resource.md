# 管理 GPU 使用

:::info
只有 Olares 管理员可以修改 GPU 模式。这样可以避免资源冲突，保证系统整体的 GPU 性能稳定可预测。
:::

Olares 让你充分发挥 GPU 的算力，加速 AI、图像和视频生成及游戏等高负载任务。你可以在 Olares 的**设置**页面，统一管理 GPU 资源。

本文将为你介绍：
- 如何选择合适的 GPU 模式。
- 如何逐步配置 GPU 模式。

## 开始之前 

配置 GPU 之前，最好先了解一下 Olares 上如何管理 GPU 绑定的。

### 应用状态与绑定

GPU 的绑定严格依赖于应用当前的状态：

- **绑定应用**：应用必须处于运行中。如果应用已停止，请先恢复运行，再将其分配给 GPU。
- **解绑应用**：应用处于运行中时，无法从 GPU 上将其移除。请先停止应用，再将其解绑。

:::info
停止应用不会自动解除该应用与 GPU 的绑定。

已停止的应用仍会显示在 GPU 应用列表中，直到你手动将其解绑。
:::

### 自动绑定

在切换 GPU 模式时，Olares 会自动将当前正在运行的 AI 应用绑定到 GPU 资源上。

绑定行为取决于所选模式：

- **时间分片**：自动绑定所有正在运行的 AI 应用，使其共享可用的 GPU 资源。
- **显存分片**：自动绑定所有正在运行的 AI 应用，并为每个应用分配运行所需的最低显存。
- **应用独占**：自动选择一个正在运行的 AI 应用，并将其绑定到某个 GPU 供其独占使用。

## 选择合适的 GPU 模式

请参考下表，根据你的工作负载，选择最适合的 GPU 模式。
| GPU 模式 | GPU 资源共享方式 | 使用场景 |
| -- | -- | -- |
| **时间分片**（默认） | 多个应用轮流使用计算资源和显存<br>来共享 GPU。 | 同时运行多个轻量级 AI 应用。 |
| **显存分片** | 将显存划分为固定配额，应用在<br>各自限额内并发运行。 | 同时运行多个 AI 应用，并精细控制每个应用可使用的 GPU 显存。 |
| **应用独占** | 单个应用独占 GPU，获得完整<br>且不中断的访问权限。 | 对性能要求极高的重负载场景，例如大语言模型或大型游戏。 |

## 查看 GPU 状态

如需查看 GPU 及其当前配置：

1. 前往**设置** > **GPU**。
2. 查看列表以获取每个 GPU 的型号、所在节点、总显存以及当前所处模式。
   ![GPU overview](/images/zh/manual/olares/gpu-overview.png#bordered){width=90%}

3. 点击某个 GPU，打开其详情页。

:::tip
如果你的设备只有一个 GPU，Olares 会直接打开该 GPU 的详情页。
:::

## 配置 GPU 模式

按照以下步骤更改 GPU 的使用方式：

1. 进入**设置** > **GPU**。
2. 点击需要配置的 GPU。
3. 在 **GPU 模式**下拉菜单选择一种模式。

:::warning 重启提示
更改 GPU 模式会解绑当前绑定到该 GPU 的应用，并重启其容器。

重启后，Olares 会自动将当前正在运行的 AI 应用重新绑定到可用的 GPU 资源。
:::

## 单 GPU 设置

如果你的 Olares 设备只有一张 GPU，请参考本节。

### 时间分片

![时间分片](/images/zh/manual/olares/gpu-time-slicing-single.png#bordered){width=90%}

#### 绑定应用

如需将应用绑定到当前 GPU：

1. 恢复已停止的应用。
   - 前往**应用市场** > **我的 Olares**，然后在下拉菜单中选择**恢复**。
   - 或前往 **设置** > **应用**，选择目标应用，然后点击**恢复**。
2. 返回**设置** > **GPU**。在**绑定应用**区域，点击**绑定应用**。
3. 选择应用，然后点击**确认**。

:::info
仅当存在运行中且尚未绑定到当前 GPU 的 AI 应用时，**绑定应用**选项才会出现。
:::

#### 解绑应用

如需将应用从当前 GPU 移除：
1. 停止应用。
   - 前往**应用商店** > **我的 Olares**，从下拉菜单中选择**暂停**。
   - 或前往**设置** > **应用**，选择目标应用后点击 **暂停**。
2. 返回**设置** > **GPU**。在**绑定应用**区域，，点击 <i class="material-symbols-outlined">link_off</i> 图标，然后点击**确认**。

### 显存分片

![显存分片](/images/zh/manual/olares/gpu-mem-slicing-single.png#bordered){width=90%}

#### 调整显存分配

1. 在**分配显存**区域找到目标应用。
2. 点击显存值旁边的 <i class="material-symbols-outlined">edit_square</i>。
3. 在**编辑 VRAM 分配**对话框中，输入所需的显存大小（GB），然后点击**确认**。

:::warning 注意
分配给所有应用的显存总量不能超过 GPU 的总显存。

如果输入的数值低于应用的最小显存需求，**确认**按钮将不可用。
:::

#### 解绑应用

如需将应用从当前 GPU 移除：

1. 停止应用。
   - 前往**应用商店** > **我的 Olares**，从下拉菜单中选择**暂停**。
   - 或前往**设置** > **应用**，选择目标应用后点击 **暂停**。
2. 返回**设置** > **GPU**。在**分配显存**区域中，点击 <i class="material-symbols-outlined">link_off</i>，然后点击 **确认**。


### 应用独占

![应用独占](/images/zh/manual/olares/gpu-app-exclusive-single.png#bordered){width=90%}

#### 切换应用

如需将当前独占应用替换为另一个应用：

1. 在**选择独占应用**区域中，点击**切换应用**。
2. 选择新的应用，然后点击**确认**。

#### 解绑应用

如需将应用从当前 GPU 移除：

1. 停止应用。
   - 前往**应用商店** > **我的 Olares**，从下拉菜单中选择**暂停**。
   - 或前往**设置** > **应用**，选择目标应用后点击 **暂停**。
2. 返回**设置** > **GPU**。在**选择独占应用**区域，点击 <i class="material-symbols-outlined">link_off</i>，然后点击**确认**。

## 多 GPU 场景

如果你的系统有多个 GPU，请参考本节。

:::info 多 GPU 调度

当系统中存在多个 GPU 时，Olares 可能会将正在运行的应用分配到不同的 GPU 上。

切换 GPU 模式后，正在运行的应用可能会显示在不同的 GPU 下。

你可以通过以下操作管理绑定关系：

- **切换GPU** 将应用移动到另一个 GPU。
- **绑定应用**将一个正在运行的应用添加到当前 GPU。
- **解绑应用**将应用从某个 GPU 上移除。

仅当你要将应用从其最后一个已绑定的 GPU 上移除时，才需要先停止应用。
:::

### 时间分片

![多GPU 时间分片](/images/zh/manual/olares/gpu-time-slicing-multi.png#bordered){width=90%}

#### 绑定应用

如果目标 GPU 上尚未绑定某应用：
1. 在**绑定应用**区域中，点击**绑定应用**。
2. 选择目标应用并点击**确认**。

#### 切换 GPU

1. 在**绑定应用**区域中找到目标应用。
2. 点击旁边的 <i class="material-symbols-outlined">repeat</i> 图标。
3. 选择目标 GPU 并点击**确认**。

在此过程中，应用将保持运行状态，不会中断。

#### 解绑应用

如需将应用从某个 GPU 上移除：

1. 如果该应用当前只绑定到这一个 GPU，请先停止应用。
   - 前往**应用商店** > **我的 Olares**，从下拉菜单中选择**暂停**。
   - 或前往**设置** > **应用**，选择目标应用后点击 **暂停**。
2. 返回**设置** > **GPU**。在**绑定应用**区域，点击 <i class="material-symbols-outlined">link_off</i>，然后点击**确认**。

:::tip 从多个 GPU 解绑
如果该应用仍绑定在同一节点的其他 GPU 上，你可以直接将其从当前 GPU 上移除，而无需停止应用。
:::

### 显存分片

![多GPU 显存分片](/images/zh/manual/olares/gpu-mem-slicing-multi.png#bordered){width=90%}

#### 调整显存分配

1. 在 **分配显存** 区域中找到目标应用。
2. 点击显存值旁边的 <i class="material-symbols-outlined">edit_square</i>。
3. 在 **Edit VRAM allocation** 对话框中，输入所需的显存大小（GB），然后点击**确认**。

:::warning 注意
分配给所有应用的显存总量不能超过 GPU 的总显存。

如果输入的数值低于应用的最小显存需求，**确认**按钮将不可用。
:::

#### 绑定应用

如果目标 GPU 当前没有绑定任何应用：

1. 在**分配显存**区域，点击**绑定应用**。
2. 选择应用，然后点击**确认**。

#### 切换 GPU

如需将应用移动到另一个 GPU：

1. 在应用列表中找到该应用，并点击其旁边的 <i class="material-symbols-outlined">repeat</i>。
2. 选择目标 GPU。
3. 点击**确认**。

#### 解绑应用

如需将应用从某个 GPU 上移除：

1. 如果该应用当前只绑定到这一个 GPU，请先停止应用。
   - 前往**应用商店** > **我的 Olares**，从下拉菜单中选择**暂停**。
   - 或前往**设置** > **应用**，选择目标应用后点击 **暂停**。
2. 返回**设置** > **GPU**。在**分配显存**区域，点击 <i class="material-symbols-outlined">link_off</i>，然后点击**确认**。

:::tip 从多个 GPU 中解绑
如果该应用仍绑定在同一节点的其他 GPU 上，你可以直接将其从当前 GPU 上移除，而无需停止应用。
:::
