# 管理加速器

Olares 使用加速器来运行 AI 推理、图像与视频生成、转码和游戏等工作负载。加速器可以是 GPU、集成加速芯片，也可以是作为兜底能力的 CPU 资源。

本文将帮助你理解 Olares 如何分配加速器，说明如何切换加速器模式、恢复已停止应用并分配资源，以及从加速器中移除应用。

## 了解加速器

### 加速器如何分配资源

Olares 会把各类加速设备（如 GPU）作为集群共享资源管理。应用不需要直接管理硬件，Olares 会根据应用的加速器需求，查找匹配的资源，并在容量足够时将应用绑定到对应资源上。

对于需要加速器的应用，Olares 在分配资源时会综合判断：

- 资源类型，例如 **NVIDIA GPU**。
- 加速器模式，例如**时间切片**、**显存切分** 或**独占**。
- 总显存与空闲显存。
- 应用显示为**单 GPU** 还是 **多 GPU**，即应用只能绑定一块加速器，还是可以同时绑定多块加速器。
- 当前已经分配到同一资源上的应用。

刚安装好的应用，如果有足够且匹配的加速器，Olares 会自动将它绑定到合适的资源上。后续如果应用停止运行，或者你想选择其他资源或模式，可以在恢复应用时手动分配资源。

### 支持的资源类型

Olares 可以展示以下加速器类型：

| 资源类型 | 说明 |
| -- | -- |
| **NVIDIA** | 通用 NVIDIA GPU。 |
| **NVIDIA GB10** | NVIDIA GB10 SoC 设备。 |
| **Apple M 系列** | Apple M 系列芯片。 |
| **AMD Strix Halo** | AMD Strix Halo 设备。 |
| **CPU** | 无可用加速设备时使用的兜底资源。 |

:::info
当前版本暂不覆盖 AMD GPU 与 AMD APU。
:::

### 加速器模式

对于支持模式切换的加速器，模式决定了应用如何共享资源。不是所有加速器都支持全部三种模式。模式下拉菜单只会显示当前加速器可用的模式。

| 模式 | 描述 | 使用场景与说明 |
| -- | -- | -- |
| **时间切片** | 多个应用随时间共享同一块<br>加速器。每个应用都可以看<br>到完整资源，但工作负载会<br>轮流使用资源。 | <ul><li>**适合**：同时运行多个需要加速器的应用。</li><li>**说明**：即使显存充足，如果所选节点的系统内存使用率达到 90% 阈值，Olares 也可能阻止启动。</li></ul> |
| **显存切分** | 多个应用共享同一块加速器，<br>并拥有固定的显存分配。 | <ul><li>**适合**：同时运行多个需要加速器的应用，并控制显存占用。</li><li>**说明**：启动应用时，Olares 默认会为应用分配它所需的最低显存，你也可以调整分配值。</li></ul> |
| **独占** | 单个应用独占加速器。 | <ul><li>**适合**：需要最大性能的重负载任务，例如大模型、渲染或高性能游戏。</li><li>**说明**：在释放该资源前，其他应用无法绑定到这块资源。</li></ul> |

## 查看加速器

前往**设置** > **加速器**，可以查看集群中所有节点上的加速器资源。

页面会展示每个可用资源的以下信息：

- 节点名称。
- 资源类型，例如 **NVIDIA GPU**。
- 加速器型号，例如 **NVIDIA GeForce RTX 4060 Ti**。
- 当前共享模式，例如**时间切片**、**显存切分**或**独占**。
- 已用显存与总显存。
- 当前分配到该资源上的应用。

![加速器概览](/images/zh/manual/olares/settings-gpu-info.png#bordered)

## 切换加速器模式

:::warning 注意
切换模式会停止当前正在使用该加速器的所有应用，并移除它们已分配的资源。如需再次使用这些应用，需要重新分配资源并启动。
:::

1. 前往**设置** > **加速器**，点击资源卡片上的**管理**。
2. 在**管理节点**页面找到目标加速器，打开模式下拉菜单，然后选择**时间切片**、**显存切分**或**独占**。

   ![选择模式](/images/zh/manual/olares/settings-gpu-dropdown.png#bordered)

## 恢复应用并分配资源

可以从**设置**或**应用商店**恢复已停止的应用，并手动分配资源。

1. 打开以下任一位置：
   - **设置** > **应用**。
   - **应用商店** > **我的 Olares**。
2. 找到目标应用，点击**恢复**。
3. 查看启动对话框顶部的应用需求，包括资源类型、资源绑定能力和显存需求。
4. 在**选择 GPU** 下选择一个**可用**资源。Olares 会自动展示符合应用需求的资源。如果某个资源无法选择，请查看提示信息：
   - **可用显存不足**：该 GPU 的总显存足够，但太多显存已分配给其他应用。展开**已分配的应用**，然后点击应用旁的**移除**，释放资源。
   - **显存不足**：该 GPU 的总显存低于应用需求。移除应用也无法解决，需要选择其他 GPU。

   ![恢复应用](/images/zh/manual/olares/settings-gpu-resume-app.png#bordered){width=70%}

5. 对于**显存切分**模式，显存默认分配为应用所需的最低值。如需调整，点击**配置**，输入显存大小，然后点击**确认**。

   ![设置显存切分模式下的显存](/images/zh/manual/olares/settings-gpu-mem-slicing-vram.png#bordered){width=70%}

   :::info 重新分配显存
   应用分配完成后，不能直接调整显存。如需稍后修改，必须先从加速器中移除该应用，再重新恢复应用。
   :::

6. 点击**启动**。

## 从加速器中移除应用

从加速器中移除应用会停止该应用，并释放它占用的硬件资源。如果你只是想从**设置**中主动释放资源，而不是立即启动其他应用，可以使用此流程。

如需稍后再次使用该应用，需要重新分配资源并启动应用。

:::warning 注意
如果应用绑定了多块加速器，或绑定了跨节点的多块加速器，从其中一个资源中移除该应用，会同时释放该应用占用的全部加速器，并停止该应用。
:::

1. 前往**设置** > **加速器**，点击资源卡片上的**管理**。
2. 在**已分配的应用**中找到目标应用，然后点击**移除**。

   ![移除应用](/images/zh/manual/olares/settings-gpu-remove-app.png#bordered)

3. 在确认对话框中点击**移除**。

:::info
资源释放需要一点时间。如果移除后仍没有可分配给该应用的资源，刷新资源列表后再查看。
:::
