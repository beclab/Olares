# 在控制面板启动或停止应用后状态不一致

如果在控制面板启动或停止应用后，应用状态与启动台、设置或商店中显示不一致，可参考本指南排查。

## 适用情况

- 在控制面板停止应用后：
  - 在**设置** > **应用**中，应用仍显示为**运行中**。
  - 在**商店** > **我的 Olares** 中，应用仍显示**打开**按钮。
  - 在启动台中，应用仍显示为可打开状态，且名称旁没有橙色暂停标识。
  - 从启动台或商店打开应用时，应用加载失败。

- 在控制面板启动应用后：
  - 在**设置** > **应用**中，应用仍显示为**暂停**。
  - 在**商店** > **我的 Olares** 中，应用仍显示**暂停**按钮。
  - 在启动台中，应用仍显示为暂停状态，名称旁有橙色标识。
  - 从启动台打开应用时，Olares 会提示你先恢复该应用。

## 原因

设置、商店和启动台通过 app-service 管理应用状态，而控制面板直接操作底层 Kubernetes 资源。

当你从控制面板启动或停止应用时，应用的工作负载会直接在 Kubernetes 层发生变化。但是，此操作不会更新由 app-service 维护并显示在设置、商店和启动台中的应用状态。因此，设置、商店和启动台中显示的应用状态会与控制面板中的状态不一致。

## 解决方案

要同步应用状态，请在设置或商店中再次执行对应操作。

以下步骤以“已在控制面板停止应用，但应用在启动台、设置或商店中仍显示为运行状态”为例。

1. 打开**设置** > **应用**，或打开**商店** > **我的 Olares**。

2. 找到受影响的应用。

3. 在设置或商店中再次暂停该应用：

   - 在设置中，点击**暂停**。
   - 在商店中，点击应用操作按钮旁边的 <i class="material-symbols-outlined">keyboard_arrow_down</i>，然后选择**暂停**。

4. 等待片刻。启动台、设置和商店中的应用状态应会与控制面板中的状态保持一致。

如果已在控制面板启动应用，但应用仍在启动台、设置或商店中显示为暂停，请按相同步骤操作，并选择**恢复**。
