# 挂载 NFS 共享

NFS（Network File System，网络文件系统）是一种广泛使用的网络文件访问协议。如果你的局域网中有 NFS 服务器，可以轻松将 NFS 共享挂载到 Olares 中，像使用本地文件夹一样访问和管理共享文件。

## 准备工作

- Olares 与 NFS 服务器必须处于同一局域网中，且网络通畅。
- 已经获取以下 NFS 共享信息：
    - **NFS 服务器 IP 地址**：例如 `192.168.1.100`。
    - **NFS 共享目录路径**：例如 `/volume1/share` 或 `/export/data`。

:::info
绝大多数家用或企业 NFS 服务器使用基于 IP 的访问控制，不需要额外的身份认证。如果你的 NFS 服务器配置了 Kerberos 等高级认证，则需要使用主机名而非 IP 地址进行挂载，且需要额外配置。本文档仅介绍最常见的无认证 NFS 挂载方式。
:::

## 将 NFS 共享挂载到 Olares

1. 打开文件管理器，进入**存储** > **外部设备**。
2. 点击右上角的**连接服务器**。
3. 在弹出的对话框中，点击**协议**下拉菜单，选择 **NFS**。
   
   ![选择 NFS 协议](/images/zh/manual/olares/files-connect-nfs-protocol.png#bordered)

4. 在**服务器地址**栏中，输入 NFS 共享地址。支持两种输入格式：
   
   - **IP 地址**：例如 `192.168.1.100`。系统会自动尝试连接该服务器并列出可挂载的共享目录供你选择。
   - **IP 地址加路径**：例如 `192.168.1.100:/volume1/share`。直接挂载指定路径，跳过目录选择步骤。

5. （可选）点击左下角的 <i class="material-symbols-outlined">add</i>，将当前服务器地址添加到**常用服务器**，方便下次快速选择并连接。

6. 点击**确定**。
   
   - 如果输入的是 IP 地址，系统会查询该服务器上所有可用的 NFS 共享目录。在**选择挂载目录**弹窗中，选择需要挂载的目录，然后点击**确认**。
   - 如果输入的是完整路径，系统会直接尝试挂载该目录。

7. 连接成功后，NFS 共享将显示在**外部设备**目录中。你可以像操作本地文件夹一样浏览和管理其中的文件。

## 查看 NFS 共享状态

由于网络波动或 NFS 服务器临时不可用，挂载的共享目录可能会在**外部设备**中显示为**不可用**，且无法进行任何文件操作。

这种情况下：
- 右键点击该目录，选择**刷新**来重新检测连接状态；或者稍后网络恢复后，系统会自动标记为可用。
- 如果目录长期不可用，建议手动卸载后重新挂载。

:::info
Olares 不会自动卸载不可用的 NFS 共享，以避免静默断开造成数据丢失。所有卸载操作均由用户手动完成。
:::

## 卸载 NFS 共享

如果你不再需要访问该 NFS 共享，可以安全地将其卸载。

1. 打开文件管理器，进入**存储** > **外部设备**。
2. 右键点击要卸载的 NFS 共享目录，然后选择**卸载**。该共享会立即与 Olares 断开连接，并从**外部设备**目录中被移除。

## 常见问题

### 为什么我输入的 IP 地址没有列出任何共享目录？

可能的原因包括：
- Olares 节点的 IP 未被 NFS 服务器允许访问。检查 NFS 服务器的 `/etc/exports` 配置，确保包含 Olares 节点的 IP 或网段。
- NFS 服务未启动或防火墙阻止了连接。请在 NFS 服务器上检查 `rpcbind` 和 `nfs-server` 服务状态。
- 服务器地址输入错误，或网络不通。

### NFS 共享需要用户名和密码吗？

通常不需要。绝大多数 NFS 部署基于 IP/主机名进行访问控制，不涉及用户名密码。如果你的环境确实配置了 Kerberos 等认证机制，则需要使用主机名（而非 IP）挂载，并提前配置 Kerberos 凭证。此类高级场景请联系系统管理员处理。

### 重启 Olares 节点后，之前挂载的 NFS 共享还会自动连接吗？

目前 Olares 会尝试在重启后自动重新挂载之前已配置的 NFS 共享。如果自动挂载失败（例如服务器未开机），该共享目录会在**外部设备**中显示为**不可用**状态，你可以稍后手动刷新或重新挂载。

### NFS 共享与 SMB 共享在使用上有什么区别？

基本操作一致，均可像本地文件夹一样管理文件。主要差异在于：
- **连接方式**：NFS 无需输入用户名密码，只需服务器地址和共享路径。
- **协议特性**：NFS 在 Unix/Linux 环境中性能更佳，文件权限与 POSIX 兼容性更好。
- **状态管理**：两者均支持不可用状态检测和前端提示，避免操作卡死。
