# GPU 应用安装或恢复后处于暂停状态

当 GPU 模式设置为**显存分片**时，如果某个依赖 GPU 的应用在安装后或恢复运行后仍处于**暂停**状态，可参考本指南进行排查。

## 适用情况

在**显存分片**模式下，出现以下任一情况：

- 安装依赖 GPU 的应用后，该应用始终处于**暂停**状态。
- 点击**恢复**后，依赖 GPU 的应用无法启动，仍处于**暂停**状态。

## 原因

在 Olares 1.12.5 中，**显存分片**模式会在应用安装或恢复运行时，自动为其分配显存。如果大部分显存已经分配给其他应用，系统就无法为目标应用提供足够的显存，导致应用无法完成初始化，从而维持在**暂停**状态。

## 解决方案：释放显存

### 步骤 1：检查应用所需显存

1. 前往**应用市场** > **我的 Olares**。
2. 点击目标应用卡片。
3. 在应用详情页中，记录该应用所需的显存。

![查看目标应用所需显存](/images/zh/manual/help/ts-mem-slice-vram-app-gpu.png#bordered){width=85%}

### 步骤 2：检查当前可用显存

1. 前往**设置** > **GPU**。
2. 在**分配显存**区域，记录列表中所有应用已分配的显存总量。
3. 用 GPU 总显存减去该数值，得到当前可用显存。

![查看当前显存分配情况](/images/zh/manual/help/ts-mem-slice-vram-gpu-mode.png#bordered){width=90%}

以上图为例，当前已分配 22 GB 显存，仅剩 2 GB 可用，而目标应用需要 4 GB，因此当前显存不足。

### 步骤 3：释放显存资源

使用以下一种或两种方式，释放足够的显存空间。

#### 减少活跃应用的显存分配

1. 在**分配显存**区域，点击某个应用显存数值旁边的 <i class="material-symbols-outlined">edit_square</i>。
2. 在不低于该应用所需显存的前提下，减少其显存分配，然后点击**确认**。

![减少显存分配](/images/zh/manual/help/ts-mem-slice-vram-reduce-vram.png#bordered){width=90%}

#### 释放暂不使用的应用资源

1. 通过以下任一方式，暂停当前不使用的应用：
   - 前往**应用市场** > **我的 Olares**，点击应用操作按钮旁的 <i class="material-symbols-outlined">keyboard_arrow_down</i>，然后点击**暂停**。
   - 或前往**设置** > **应用**，选择该应用，然后点击**暂停**。
2. 返回**设置** > **GPU**。
3. 在**分配显存**区域，点击已暂停应用旁边的 <i class="material-symbols-outlined">link_off</i>，然后点击**确认**。
4. 重复以上步骤，直到有足够显存可用。

### 步骤 4：恢复目标应用

1. 通过以下任一方式，恢复目标应用：
   - 在**应用市场** > **我的 Olares** 中，点击应用操作按钮旁边的 <i class="material-symbols-outlined">keyboard_arrow_down</i>，然后点击**恢复**。
   - 或前往**设置** > **应用**，选择该应用后点击**恢复**。
2. 等待应用状态变为**运行中**。

![释放显存后恢复目标应用](/images/zh/manual/help/ts-mem-slice-vram-resume-outcome.png#bordered){width=90%}
