# 统一应用网关（app-gateway）

## 概述

**app-gateway** 是 Olares 平台在集群内的**统一应用访问入口**。用户与工作负载通过一致的访问地址（基于 Host 域名）访问 Shared 应用与系统应用，无需记忆各服务分散的地址，也不能在集群内绕过平台直接访问后端服务。

网关基于 **Gateway API** 与 **Envoy** 数据面实现，与平台服务网格协同，为集群内、经平台收敛的南北向流量提供转发能力。

## 主要能力


| 能力       | 说明                                                 |
| -------- | -------------------------------------------------- |
| 统一入口     | 按访问域名（Host）识别目标应用，路径保持与原应用一致，避免破坏前端与 API 路径约定      |
| 集群内访问治理  | 业务 Pod 经网关访问受管应用，配合平台身份与服务网格，减少无身份直连               |
| 与外部入口协同  | 公网、局域网、VPN 等仍由平台既有 L4 入口承接；集群内流量经 app-gateway 统一纳管 |
| 可扩展路由与策略 | 支持按应用、入口维度发布与调整路由，并与平台安装、升级流程衔接                    |


## 工作原理（简要）

1. 客户端（用户浏览器、集群内其他应用等）使用平台分配的**统一访问地址**发起请求。
2. 请求到达 **app-gateway**，按 Host 匹配目标应用与入口。
3. 经平台鉴权与策略判定通过后，网关将请求转发至对应应用服务。
4. 访问过程可纳入平台审计与可观测体系。

```text
客户端 → app-gateway → 目标应用服务
              ↑
        平台鉴权与策略
```

## 部署与运行

- app-gateway 随 **Olares 平台安装或升级**自动部署，默认运行在 `**app-gateway`** 命名空间。  
- 平台管理员一般**无需单独安装或手工维护**网关组件；容量与版本随平台版本一并交付。  
- 与 **Linkerd 服务网格**配合使用时，集群内流量可获得传输层身份保护与相关可观测能力。

## 适用场景

- 多租户、多应用环境下，需要**统一域名与统一入口**访问 Shared 与系统应用。  
- 需要满足「集群内访问必须经过平台入口与权限体系」的安全与合规要求。  
- 应用安装、升级、灰度时，希望入口与路由随平台声明自动生效。

## 相关组件

以下能力由平台其他模块与网关协同完成，共同构成完整统一访问体验：


| 组件       | 面向用户的作用                           |
| -------- | --------------------------------- |
| 平台鉴权服务   | 对访问请求进行允许/拒绝判定，并驱动策略更新            |
| 应用生命周期服务 | 应用安装、升级时同步入口、路由与授权相关信息            |
| 服务网格     | 为集群内调用提供安全传输与调用方可观测性              |
| 外部 L4 入口 | 承接来自公网与边缘网络的 HTTPS 访问，并与集群内网关路径衔接 |


## 版本与支持

app-gateway 的能力与行为以当前 Olares 发行说明及平台文档为准。升级平台时，网关组件将随平台一并升级；请勿单独替换或混用未在发行说明中列出的第三方网关版本。
